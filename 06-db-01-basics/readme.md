# Домашнее задание к занятию "6.1. Типы и структура СУБД"

## Задача 1

Архитектор ПО решил проконсультироваться у вас, какой тип БД 
лучше выбрать для хранения определенных данных.

Он вам предоставил следующие типы сущностей, которые нужно будет хранить в БД:

- Электронные чеки в json виде
- Склады и автомобильные дороги для логистической компании
- Генеалогические деревья
- Кэш идентификаторов клиентов с ограниченным временем жизни для движка аутенфикации
- Отношения клиент-покупка для интернет-магазина

Выберите подходящие типы СУБД для каждой сущности и объясните свой выбор.

### _Ответ:_

- Электронные чеки в json виде: Документо-ориентированная

- Склады и автомобильные дороги для логистической компании: Сетевая - иерархическая с множеством узлов, Графовые СУБД

- Генеалогические деревья: Иерархическая- класические деревья с одним родителем

- Кэш идентификаторов клиентов с ограниченным временем жизни для движка аутенфикации: Ключ-значение, скорость доступа, простота

- Отношения клиент-покупка для интернет-магазина - На стадии стартапа или MVP - использовать NoSQl, когда проект "утресётся" перевести на реляционную БД
